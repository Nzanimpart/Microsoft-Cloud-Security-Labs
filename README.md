## 🛡️ Project 1: Zero-Trust Architecture & Identity Management
**Objective:** Designing a secure identity perimeter using Microsoft Entra ID (formerly Azure Active Directory) principles.

### Key Implementation Concepts:
* **Explicit Verification:** Always authenticate and authorize based on all available data points, including user identity, location, device health, service or workload, data classification, and anomalies.
* **Least Privileged Access:** Limiting user access with Just-In-Time (JIT) and Just-Enough-Access (JEA), risk-based adaptive policies, and data protection to protect both data and productivity.
* **Assume Breach:** Minimizing blast radius and segmenting access. Utilizing end-to-end encryption and continuous monitoring to detect and respond to threats.

### Simulated Conditional Access Policy Workflow:
1. **Condition:** User attempts to log into the Azure Portal from an untrusted, external IP address.
2. **Control:** System automatically triggers an Multi-Factor Authentication (MFA) prompt via the Microsoft Authenticator app.
3. **Session Risk Assessment:** If the device is unmanaged or uncompliant, file downloads are blocked entirely during the cloud session.

## ⚖️ Project 2: Cloud Compliance & Governance Frameworks
**Objective:** Ensuring cloud infrastructure adheres to international regulatory frameworks (GDPR, ISO 27001) using Microsoft Purview and Microsoft Defender for Cloud.

### Core Framework Structures Documented:
* **Shared Responsibility Model:** Mapping security boundaries between the Cloud Service Provider (Microsoft) and the Customer across IaaS, PaaS, and SaaS environments.
* **Data Classification Matrices:** Establishing labels (Public, General, Confidential, Highly Sensitive) to prevent Data Loss Prevention (DLP) incidents.
* **Microsoft Purview Integration:** Tracking sensitive information types (such as passport numbers or credit cards) to prevent unauthorized external sharing.
